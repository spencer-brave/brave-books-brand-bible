Section 01

How This System Works

Every product in the BRAVE Books catalog passes through four classification layers. Each layer serves a different purpose and is managed independently. Understanding what each layer does — and what it doesn’t do — is the foundation of this SOP.

Layer 1
Product Category
Tax & channels

Shopify’s predefined taxonomy. Handles tax rates and Google/Facebook channel sync. Set once per product. Rarely changes.

Layer 2
Product Type
What is it?

A single canonical label per product. Powers the first tier of automated collections. Choose from the approved list — one value per product.

Layer 3
Product Tags
How do we find it?

Multiple labels per product. Powers smart collections, storefront filters, and internal operations. Follow the prefix convention exactly.

Layer 4
Metafields
Rich display data

Structured key-value fields for frontend display: author names, age ranges, book topics, cross-sell links. Not for filtering — tags handle that.

Key principle: Tags power collections and filters. Metafields power the frontend display. Product Type is the structural backbone. Product Category is for Shopify’s systems. Never use one layer to do another layer’s job.

Section 04

Product Tags

Tags are the engine that powers smart collections, storefront filters, and internal organization. Every tag follows a strict prefix convention to prevent collisions and keep the system scannable.

Tag Format Rules

RuleDetails
CharactersLowercase letters, numbers, and hyphens only. No spaces, no special characters, no uppercase.
Structureprefix-value format. The prefix identifies the tag’s purpose. The value is the specific item.
LengthKeep under 40 characters. Shorter is better.
ConsistencyUse the exact tag from the master list. Don’t paraphrase (age-3to5 is wrong, age-3-5 is right).
One per conceptDon’t tag “courage” AND “bravery” for the same concept. Pick the canonical value.
Customer visibilitySome tags surface in storefront filter menus. Use values that make sense to parents, not internal jargon.

Section 06

New Product Entry Checklist

Use this checklist every time a new product is added to Shopify. Walk through each step in order. This ensures every product is fully classified before it goes live.

1
Set Product Category
Choose the correct Shopify Standard Taxonomy category from the approved list in Section 2. If unsure, default to Media > Books > Print Books for any book product.
2
Set Product Type
Choose exactly one Product Type from the approved list in Section 3. Do not create new types without updating this SOP.
3
Add author- tag
Look up the author in the master tag list. New author? Use author-firstname-lastname (lowercase, hyphens, no titles). For BRAVE Books house products, use author-brave-books.
4
Add age- tag(s)
Determine the recommended reading age. Most picture books are age-3-5. Board books are age-0-2. Apply multiple if the book spans ranges.
5
Add value- tag(s)
Identify 1–3 moral values/lessons from the approved list. The primary lesson should be the first value tag applied.
6
Add series- tag
Freedom Island? series-freedom-island + appropriate saga- tag. Black Heart? series-black-heart. Everything else: series-standalone.
7
Add format- tag
format-hardcover, format-board-book, or format-paperback. Most BRAVE titles are format-hardcover.
8
Add status- tag(s)
New launch? status-new-release (remove after 60 days). Pre-order? status-pre-order (remove when shipping). Seasonal? status-seasonal.
9
Add line- tag (non-books only)
Plushes: line-plush. Games: line-game. Activity products: line-activity. Books don’t need a line- tag.
10
Add bundle-type- tag (bundles only)
If Product Type is Bundle, add exactly one bundle-type- tag to classify what kind of bundle.
11
Add ops- tags if needed
Exclude from upsells? ops-upsell-exclude. Hidden from SEO? ops-seo-hidden. Subscription-eligible? ops-subscription-eligible.
12
Fill metafields
Author Name (handle), Author Display Name, Author Bio, Book Lesson, Book Topics, Short Description. For Freedom Island books: Saga Number and Book Number.
13
Set Related + Complementary Products
Add 3–6 related product handles and 1–3 complementary product handles via the metafield editor.
14
Verify
Spot-check: Does this product appear in the correct smart collections? Walk the nav tree and confirm it shows up where expected.

Speed matters: With ~30 launches in 2026, you’ll be adding products every 2 weeks. This checklist should take under 10 minutes per product once familiar. The time investment pays off in zero manual collection management — smart collections handle everything automatically.

Tag Migration Plan

The current tag system has 250+ unique tags with no naming convention. Many are redundant, inconsistently cased, or meaningless to customers. The migration plan:

1
Export all products via CSV
Use Shopify’s bulk editor or a CSV export/import cycle.
2
Clear all existing tags
Wipe the slate. The old tags are unsalvageable as a system.
3
Re-tag using the new prefix convention
Walk through each product using the Section 6 checklist.
4
Build smart collections
Create automated collections using the new tag/type conditions.
5
Wire up the navigation
Link the new collections to the mega menu and nav structure.
6
Verify
Spot-check every collection page to confirm correct products appear.

How to use this document: Print it or keep it open alongside Shopify Admin. Every time a product is added, edited, or audited, reference Sections 2–6. If a decision doesn’t fit neatly into this system, flag it — don’t improvise. Consistency is more valuable than speed.
